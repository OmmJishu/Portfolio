ToDoList Application

Developed a user authentication Console based ToDo List software in C++ for task management to enhance productivity.

Tools & Technologies Used: C++, STL, Visual Studio, Database: File System, Data Structures: vector, unorderedset, Lambda Expressions

Key Points:

  • Task Management: The ToDo List Application provides a convenient way to manage tasks by allowing users to add, remove, modify, and view tasks.
  • User Authentication: Users are required to log in with a unique ID and password. New users can register by creating their ID and password, which are then stored securely for future logins.
  • Task Attributes: Each task in the ToDo list has three main attributes: Description, Due Date, and Priority.
  • Persistent Storage: User-specific task lists are stored in text files named after the user's ID and password combination.
  • Functionality: The application includes features like adding, removing, modifying, and viewing tasks, as well as saving changes and exiting.
  • Error Handling: The application includes error handling mechanisms to handle scenarios such as incorrect user IDs or passwords, invalid task descriptions, and file access errors.
  • User-Friendly Interface: The application provides a user-friendly interface with clear prompts and instructions for each action.
  • Security: User authentication and task data storage are implemented with security in mind to protect user privacy and prevent unauthorized access to task lists.
  • Customization: Users can customize their task lists by adding tasks according to their needs and preferences.

Tic-Tac-Toe Game

Implemented a console-based Tic-Tac-Toe game application in C++, providing an interactive platform for two players to compete against each other.

Tools & Technologies Used: C++, STL, Visual Studio

Key Points:

  • Classic Game: Tic-Tac-Toe is a classic two-player game where players take turns marking spaces in a 3x3 grid to achieve a winning pattern.
  • Console-Based Interface: The game features a simple console-based interface that allows players to interact using keyboard inputs.
  • Player Versus Player: The main gameplay mode enables two human players to compete against each other, taking turns to make moves.
  • Winning Conditions: The game checks for winning conditions after each move to determine if a player has achieved a winning pattern, such as three in a row, column, or diagonal.
  • Draw Detection: In case all spaces are filled without a winner, the game detects a draw and ends the match.
  • User-Friendly Experience: The game provides clear prompts and instructions for players, making it easy to understand and play.
  • Error Handling: Error handling mechanisms are implemented to handle invalid inputs and prevent unexpected behavior during gameplay.
  • Scalability: The code is designed to be scalable, allowing for potential future enhancements such as AI opponents or additional gameplay modes.
  • Educational Purpose: The project serves as an educational tool for learning basic game development concepts and practicing C++ programming skills.
